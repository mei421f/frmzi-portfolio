# پورتفولیوی معین فرامرزی — با پنل مدیریت

سایت پورتفولیوی شخصی با بک‌اند Node.js/Express که امکان اضافه‌کردن نمونه‌کار جدید (آپلود
مستقیم فایل ویدیو + عنوان و توضیح) از طریق یک پنل مدیریت با رمز عبور را فراهم می‌کند.

## ساختار پروژه

```
frmzi-portfolio/
├── backend/          ← سرور Express (API + آپلود فایل + پنل مدیریت)
│   └── data/
│       ├── projects.json   ← اطلاعات نمونه‌کارها
│       └── videos/         ← فایل‌های ویدیوی آپلودشده
└── public/            ← فرانت (HTML/CSS/JS ساده، بدون نیاز به build)
    ├── index.html      ← صفحه‌ی اصلی سایت
    ├── admin.html       ← پنل مدیریت
    └── assets/
```

## پیش‌نیاز

Node.js نسخه ۱۸ به بالا.

## راه‌اندازی لوکال

```bash
cd backend
npm install
```

فایل `backend/.env` را باز کن و این دو مقدار را عوض کن:

```
JWT_SECRET=یک-رشته-تصادفی-طولانی
ADMIN_PASSWORD=یک-رمز-عبور-برای-خودت
```

سپس:

```bash
npm run dev
```

- سایت: `http://localhost:5000`
- پنل مدیریت: `http://localhost:5000/admin`

با همان `ADMIN_PASSWORD` که در `.env` گذاشتی وارد پنل شو، نمونه‌کار جدید اضافه کن؛ همان لحظه
در صفحه‌ی اصلی سایت (بخش «نمونه‌کارها») نمایش داده می‌شود.

## دیپلوی روی Railway

### ۱. پروژه را روی GitHub بگذار و به Railway وصل کن
- ریپازیتوری بساز، پوشه‌ی `frmzi-portfolio` را push کن.
- در [railway.com](https://railway.com): `New Project` → `Deploy from GitHub repo`.

### ۲. متغیرهای محیطی را تنظیم کن
در تب `Variables` سرویس:
```
JWT_SECRET=یک-رشته-تصادفی-طولانی
ADMIN_PASSWORD=رمز-عبور-پنل-مدیریت-تو
DATA_DIR=/data
```
(نیازی به تنظیم دستی `PORT` نیست، Railway خودش تزریق می‌کند.)

### ۳. یک Volume بساز — این مرحله حیاتی است
چون فایل‌های ویدیوی آپلودشده باید بین دیپلوی‌های بعدی از بین نروند، باید یک Volume دائمی
وصل کنی:
- در سرویس اصلی → تب `Volumes` → `+ New Volume`
- مسیر Mount را دقیقاً بگذار: `/data`

این دقیقاً همان مسیری است که در متغیر `DATA_DIR` تعریف کردیم، پس فایل‌های آپلودشده و
`projects.json` همیشه روی این فضای دائمی ذخیره می‌شوند و با دیپلوی مجدد پاک نمی‌شوند.

### ۴. دامنه بساز
تب `Settings → Networking → Generate Domain`.

بعد از بالا آمدن سایت، پنل مدیریت روی آدرس `https://دامنه‌ات/admin` در دسترس است.

## نکته امنیتی

- حتماً `ADMIN_PASSWORD` و `JWT_SECRET` را قبل از انتشار عمومی به مقادیر قوی و اختصاصی تغییر بده.
- پنل مدیریت فقط با یک رمز عبور محافظت می‌شود (چون سایت تک‌کاربره است)؛ رمز را جایی امن نگه دار
  و در صورت لو رفتن، همان مقدار را در Variables ریلوی عوض و سرویس را ری‌استارت کن.
- محدودیت حجم آپلود فعلاً ۱۵۰ مگابایت روی هر فایل است (در `backend/routes/projects.js` قابل تغییر).
